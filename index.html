Get Start

一，github静态页创建

1. 创建github帐号

2. 创建page仓库

github 为每个人提供了一个创建静态博客的仓库,新建仓库，命名为 xxx.github.io 如 leinov.github.io, 只有这种格式命名的仓库才可以被访问。

将刚创建的仓库克隆下来， 如果没有安装git，需要安装，这里有一个完整的git教程 git教程

3. 创建页面并push

克隆下来的项目暂时还是个空的文件

创建文件index.html 在index.html 输入 this is my github page

将仓库推向远端 会提示输入username喝password 输入github的用户名和密码即可 也可以添加ssh省略此步输入

新创建的文件就被推送到了远端仓库 访问 http://yourname.github.io 这样一个gitbub个站就完成了.

二， HEXO

1.hexo介绍

A fast ,simple & powerful blog framework .一个快速，简单，强大的博客框架，hexo是node开发的，所以你的先安装node

2.安装

hexo的安装符合他的标语，fast，simple 确实快速简单,以下几个命令就可以完成整个操作

安装全局的hexo命令

初始化一个blog

进入blog

安装node依赖包

启动服务

在浏览器里访问 http://localhost:4000 或者 http://0.0.0.0:4000/ 如果出线下面的页面显示 恭喜你 hexo安装完成啦 是不是很简单!

Tip 如果在hexo server 时报错 安装server包就可以了

3.建立新的博客

创建新的markdown博客source并生成静态文件 刷新http://localhost:4000 或 http://0.0.0.0:4000/ 查看

4. hexo文件结构

  • blog是我们的博客文件夹
  • node_modules里是系统所依赖的所有node包，
  • public是静态文件存放文件夹，在执行 hexo generat 时会生成该文件夹以及下面文件，其实访问的页面就是在这个文件夹下
  • scaffolds 模版 文件夹。当您新建文章时，Hexo 会根据 scaffold 来建立文件。
  • source 存放用户资源，也就是我们编辑的markdown文件，他将会宝贝解析到public
  • themes 存放主题，网站的样式
